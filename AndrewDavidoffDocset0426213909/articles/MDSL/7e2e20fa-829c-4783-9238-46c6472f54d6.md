 <b>BitmapCreateOptionsSyntax</b> </td>
 </tr>
 <tr><td><div class="indent0">[values]</div></td>
 <td></td>
 </tr>
 <tr><td><div class="indent2">BackgroundCreation (5)</div></td>
 <td>Causes a BitmapSource to be initialized as soon as it is declared. This option uses the image cache for previously used URIs. If an image is not in the image cache, the image will be downloaded and decoded on a separate background thread.</td>
 </tr>
 <tr><td><div class="indent2">DelayCreation</div></td>
 <td>Causes a BitmapSource object to delay initialization until it is necessary. This is useful when dealing with collections of images. This is the default value of the CreateOptions property in Silverlight and Silverlight for Windows Phone.</td>
 </tr>
 <tr><td><div class="indent2">IgnoreImageCache</div></td>
 <td>Initializes images without using an existing image cache. Any existing entries in the image cache are replaced, even if they share the same URI. This option should only be selected when images in a cache need to be refreshed.</td>
 </tr>
 <tr><td><div class="indent2">None</div></td>
 <td>No initialization options are specified. This is the NOT the default value for the CreateOptions property in Silverlight or Silverlight for Windows Phone (DelayCreation is the default).</td>
 </tr>
